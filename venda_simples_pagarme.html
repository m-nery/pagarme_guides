Guia de Integração: Venda Simples com Cartão

Um guia simples do fluxo da API Pagar.me para uma transação com cartão de crédito.

Para detalhes completos sobre cada endpoint, consulte a documentação oficial da Pagar.me.

1

Cadastro do Cliente

O primeiro passo é cadastrar o cliente que fará a compra. O `customer_id` retornado será usado nas próximas etapas.

Link da documentação: https://docs.pagar.me/reference/criar-cliente-1

curl --request POST \ --url https://api.pagar.me/core/v5/customers \ --header 'Authorization: Basic c2tfdGVzdF9zaW1wbGVleGFtcGxlOg==' \ --header 'Content-Type: application/json' \ --data ' { "name": "Tony Stark", "email": "avengerstark@starkindustries.com.br", "document": "87232965001", "type": "individual", "address": { "line_1": "375, Av. General Justo, Centro", "line_2": "8º andar", "zip_code": "20021130", "city": "Rio de Janeiro", "state": "RJ", "country": "BR" }, "phones": { "home_phone": { "country_code": "55", "number": "22180513", "area_code": "21" } } } '
{ "id": "cus_xxxxxxxxxxxxxxxx", "name": "Tony Stark", "status": "active" }
2

Criação do Token do Cartão

Esta etapa é executada no front-end da sua aplicação (por exemplo, no navegador do cliente), usando sua chave pública (`public_key`) para garantir que os dados sensíveis do cartão não passem pelo seu servidor.

O card token gerado tem validade de um minuto e é limitado a um uso único.

Link da documentação: https://docs.pagar.me/reference/criar-token-cartão-1

curl --request POST \ --url https://api.pagar.me/core/v5/tokens \ --header 'Authorization: Basic cGtfdGVzdF9zaW1wbGVleGFtcGxlOg==' \ --header 'Content-Type: application/json' \ --data ' { "type": "card", "card": { "number": "4242424242424242", "holder_name": "Tony Stark", "exp_month": 12, "exp_year": 2030, "cvv": "123" } } '
{ "id": "token_xxxxxxxxxxxxxxxx", "type": "card" }
3

Criação do Cartão a partir do Token

No seu back-end, use o `customer_id` (do Passo 1) e o `token` (do Passo 2) para salvar o cartão de forma segura no cofre da Pagar.me.

Link da documentação: https://docs.pagar.me/reference/criar-cartao

curl --request POST \ --url https://api.pagar.me/core/v5/customers/cus_xxxxxxxxxxxxxxxx/cards \ --header 'Authorization: Basic c2tfdGVzdF9zaW1wbGVleGFtcGxlOg==' \ --header 'Content-Type: application/json' \ --data ' { "token": "token_xxxxxxxxxxxxxxxx", "billing_address": { "line_1": "375, Av. General Osorio, Centro", "line_2": "7º Andar", "zip_code": "220000111", "city": "Rio de Janeiro", "state": "RJ", "country": "BR" } } '
{ "id": "card_xxxxxxxxxxxxxxxx", "last_four_digits": "4242", "status": "active" }
4

Criação do Pedido

Finalmente, crie o pedido usando o `customer_id` e o `card_id` salvos. A opção `"capture": true` faz com que a cobrança seja efetuada imediatamente.

Link da documentação: https://docs.pagar.me/reference/criar-pedido

curl --request POST \ --url https://api.pagar.me/core/v5/orders \ --header 'Authorization: Basic c2tfdGVzdF9zaW1wbGVleGFtcGxlOg==' \ --header 'Content-Type: application/json' \ --data ' { "customer_id": "cus_xxxxxxxxxxxxxxxx", "items": [ { "amount": 15000, "description": "Item de Teste", "quantity": 1, "code": "ITEM001" } ], "payments": [ { "payment_method": "credit_card", "credit_card": { "installments": 1, "capture": true, "card_id": "card_xxxxxxxxxxxxxxxx" } } ] } '
{ "id": "or_xxxxxxxxxxxxxxxx", "status": "paid", "charges": [ { "id": "ch_xxxxxxxxxxxxxxxx", "amount": 15000, "status": "paid" } ] }
5

Cancelamento da Cobrança

Por fim, se necessário, cancele a cobrança (estorno) usando o `charge_id` retornado no passo anterior. É possível realizar um estorno parcial (enviando o `amount`) ou total (omitindo o `amount`).

Link da documentação: https://docs.pagar.me/reference/cancelar-uma-cobrança

curl --request DELETE \ --url https://api.pagar.me/core/v5/charges/ch_xxxxxxxxxxxxxxxx \ --header 'Authorization: Basic c2tfdGVzdF9zaW1wbGVleGFtcGxlOg==' \ --header 'Content-Type: application/json' \ --data ' { "amount": 15000 } '
{ "id": "ch_xxxxxxxxxxxxxxxx", "amount": 15000, "status": "canceled" }
