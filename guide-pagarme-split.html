Guia de Integração: Venda com Split de Pagamento

Um guia simples do fluxo da API Pagar.me para uma transação com divisão de valores.

Para detalhes completos sobre cada endpoint, consulte a documentação oficial da Pagar.me.

1

Criação dos Recebedores

O primeiro passo é cadastrar os recebedores (sellers) que participarão da divisão de valores. Sua conta Pagar.me já possui um recebedor principal que será usado para representar o marketplace na regra de Split. Você pode verificar esse recebedor na dashboard Pagar.me no menu Recebedores. O recipient_id` retornado será usado na criação do pedido. Uma observação importante: o documento do recebedor deverá ser igual ao documento da conta bancária, ou seja, só será permito transfência de valores para contas bancárias vinculadas ao CPF/CNPJO do recebedor.

2

Cadastro do Cliente

O próximo passo é cadastrar o cliente que fará a compra. O `customer_id` retornado será usado nas próximas etapas.

POST https://api.pagar.me/core/v5/customers { "name": "Steve Rogers", "email": "captain@avengers.com", "document": "27256221034", "type": "individual", "address": { "line_1": "569, Lefferts Place, Brooklyn", "line_2": "Apt 4", "zip_code": "11238", "city": "New York", "state": "NY", "country": "US" }, "phones": { "home_phone": { "country_code": "1", "number": "5551234", "area_code": "718" } } }
{ "id": "cus_xxxxxxxxxxxxxxxx", "name": "Steve Rogers", "status": "active" }
3

Criação do Token do Cartão

Esta etapa é executada no front-end da sua aplicação (por exemplo, no navegador do cliente), usando sua chave pública (`public_key`) para garantir que os dados sensíveis do cartão não passem pelo seu servidor.

POST https://api.pagar.me/core/v5/tokens { "type": "card", "card": { "number": "4242424242424242", "holder_name": "Steve Rogers", "exp_month": 12, "exp_year": 2030, "cvv": "123" } }
{ "id": "token_xxxxxxxxxxxxxxxx", "type": "card" }
4

Criação do Cartão a partir do Token

No seu back-end, use o `customer_id` (do Passo 2) e o `token` (do Passo 3) para salvar o cartão de forma segura no cofre da Pagar.me.

POST https://api.pagar.me/core/v5/customers/cus_xxxxxxxxxxxxxxxx/cards { "token": "token_xxxxxxxxxxxxxxxx" }
{ "id": "card_xxxxxxxxxxxxxxxx", "last_four_digits": "4242", "status": "active" }
5

Criação de um Pedido com Split

Ao criar o pedido, inclua o objeto `split` para definir como o valor da transação será dividido entre os recebedores (o seu marketplace e o seller).

POST https://api.pagar.me/core/v5/orders { "customer_id": "cus_xxxxxxxxxxxxxxxx", "items": [ { "amount": 20000, "description": "Item com Split", "quantity": 1, "code": "ITEM-SPLIT-01" } ], "payments": [ { "payment_method": "credit_card", "credit_card": { "installments": 1, "capture": true, "card_id": "card_xxxxxxxxxxxxxxxx" }, "split": [ { "amount": 18000, "recipient_id": "re_xxxxxxxxxxxxxxxx", "type": "flat", "options": { "charge_processing_fee": true, "charge_remainder_fee": true, "liable": true } } ] } ] }
{ "id": "or_xxxxxxxxxxxxxxxx", "status": "paid", "charges": [ { "id": "ch_xxxxxxxxxxxxxxxx", "amount": 20000, "status": "paid" } ] }
6

Cancelamento da Cobrança

Por fim, se necessário, cancele a cobrança (estorno) usando o `charge_id` retornado no passo anterior. É possível realizar um estorno parcial (enviando o `amount`) ou total (omitindo o `amount`).

DELETE https://api.pagar.me/core/v5/charges/ch_xxxxxxxxxxxxxxxx { "amount": 20000 }
{ "id": "ch_xxxxxxxxxxxxxxxx", "amount": 20000, "status": "canceled" }
